# v4.1 Binding Ownership Refactor — Implementation Plan

> Baseline: `docs/baseline/engine-architecture-baseline.md` (v4.1 amendment).
> Decisions: AD55–AD58.
>
> External API boundary: `docs/user-facing-api.md` (v0.2).

## Goal

Implement the four v4.1 binding-ownership decisions:

1. **AD55**: Remove `ProtocolBinding` facade.
2. **AD56**: `ServerBinding` gains explicit `serve`/`shutdown` lifecycle;
   `configure`/`register_thing`/`unregister_thing`/`set_event_broker`/
   `set_request_sink` merged or removed; `FanInSender` deleted.
3. **AD57**: Remove `ClientBindingFactory`; `ClientBinding` stored as shared
   `Arc`.
4. **AD58**: Binding ownership moves to handles; Servient retains defaults.

## Scope

Three crates touched, sequenced bottom-up along the dependency graph. Each
step leaves its target crate compiling in isolation; the workspace is whole
again at Step 4.

| Step | Crate | Focus |
|---|---|---|
| 1 | `clinkz-wot-core` | Trait rewrites: `ServerBinding` lifecycle; remove `ClientBindingFactory`, `ProtocolBinding`, `FanInSender`; `ConsumedThing` Arc bindings. |
| 2 | `clinkz-wot-servient` | Servient shape; `ServientBuilder` API; handle ownership; `expose`/`destroy` → `serve`/`shutdown`. |
| 3 | `clinkz-wot-protocol-bindings-zenoh` | `ZenohServerBinding` lifecycle; replace `ZenohProtocolBinding` with direct constructors; remove `ZenohClientFactory`. |
| 4 | Workspace | Integration tests; clippy; no-std check; verify end-to-end. |

## Entry Criteria

- v4.1 baseline amendment is locked (AD55–AD58).
- `docs/user-facing-api.md` v0.2 reflects the target surface.

## Exit Criteria

- `cargo check --workspace --all-targets` passes.
- `cargo test --workspace` passes.
- `cargo clippy --workspace --all-targets` passes.
- `scripts/check-no-std.sh` passes for core crate root.
- `ExposedThingHandle::expose` calls `ServerBinding::serve` and starts the
  zenoh driving task (the dead `async_rx` gap is closed).
- `ConsumedThingHandle` holds `Arc<dyn ClientBinding>` (shared, no factory).

## Step 1 — Core trait rewrites (`clinkz-wot-core`)

### 1.1 Rewrite `ServerBinding` trait (`core/src/inbound.rs`)

Replace the v4.0 trait surface:

```rust
// REMOVE:
fn configure(&self, _ctx: &BindingContext) {}
fn register_thing(&self, thing_id: &ThingId, td: &Thing) -> Result<(), CoreError>;
fn unregister_thing(&self, thing_id: &ThingId);

// ADD:
fn serve(&self, thing_id: &ThingId, td: &Thing, ctx: &BindingContext) -> CoreResult<()>;
fn shutdown(&self, thing_id: &ThingId);

// KEEP (unchanged):
fn try_accept(&self) -> Option<InboundRequest> { None }
fn send_response(&self, response: InboundResponse);
```

### 1.2 Simplify `BindingContext` (`core/src/inbound.rs`)

```rust
// REMOVE:
#[cfg(feature = "std")]
pub fanin_sender: Option<FanInSender<InboundRequest>>,

// KEEP:
pub event_broker: EventBroker,
#[cfg(feature = "async")]
pub dispatch: Option<Arc<dyn Dispatch>>,
```

### 1.3 Remove `FanInSender` (`core/src/inbound.rs`)

Delete the `FanInSender<T>` type alias and its `pub use` in `lib.rs`.

### 1.4 Remove `ClientBindingFactory` (`core/src/binding.rs`)

Delete the `ClientBindingFactory` trait and its `pub use` in `lib.rs`.

### 1.5 Delete `binding_facade` module

Remove `core/src/binding_facade.rs` entirely. Remove its `mod` declaration
and `pub use` re-exports from `lib.rs`:
- `ProtocolBinding`, `ProtocolId`, `ClientOnly`, `ServerOnly`, `client_only`,
  `server_only`.

### 1.6 Change `ConsumedThing::bindings` to `Arc`

In `core/src/thing.rs`:

```rust
// BEFORE:
pub(crate) bindings: Vec<Box<dyn ClientBinding>>,
pub fn register_binding(&mut self, binding: impl ClientBinding + 'static) {
    self.bindings.push(Box::new(binding));
}

// AFTER:
pub(crate) bindings: Vec<Arc<dyn ClientBinding>>,
pub fn register_binding(&mut self, binding: Arc<dyn ClientBinding>) {
    self.bindings.push(binding);
}
```

Also update the `ClientBinding for Box<dyn ClientBinding>` impl in
`binding.rs` — either remove it (no longer needed) or add an
`Arc<dyn ClientBinding>` equivalent if dispatch through `Arc` requires it.
Check whether `bindings.iter().find(...)` works with `Arc<dyn ClientBinding>`
directly (it should — `Arc<T>` derefs to `T`).

### 1.7 Update core unit tests

- Remove `binding_facade.rs` tests (`client_only_*`, `server_only_*`,
  `protocol_id_*`).
- Update `inbound.rs` test stubs (`NoopServer`) to implement `serve`/
  `shutdown` instead of `register_thing`/`unregister_thing`/`configure`.
- Update any `ClientBindingFactory` references in test modules.

// AFTER:
pub fn with_server_binding(mut self, binding: Arc<dyn ServerBinding>) -> Self {
    self.server_bindings.push(binding);
    self

    self.client_bindings.push(binding);

- Remove `BoxedFactory` adapter entirely.
- Remove the `configure(&ctx)` loop in `build()` — `serve()` is now called
  per-Thing during `expose()`, not once globally at build time.

### 2.3 Update `ExposedThingHandle` (`servient/src/handle.rs`)

Add server-bindings ownership:

```rust
pub struct ExposedThingHandle {
    servient: Servient,
    slot: Arc<WotLock<ExposedThingSlot>>,
    id: ThingId,
    server_bindings: Arc<[Arc<dyn ServerBinding>]>,  // NEW
}
```

- `Servient::produce()` clones `default_server_bindings` into the handle.
- `expose()` passes the handle's bindings to `Servient::expose_thing()`.
- `destroy()` passes them to `Servient::destroy_thing()`.

### 2.4 Update `Servient::expose_thing` (`servient/src/servient.rs`)

```rust
// BEFORE:
for (i, binding) in self.server_bindings.iter().enumerate() {
    binding.register_thing(&id, &td)?;
}

// AFTER:
for (i, binding) in bindings.iter().enumerate() {
    let ctx = BindingContext {
        event_broker: self.event_broker.clone(),
        dispatch: Some(Arc::new(self.clone())),
    };
    binding.serve(&id, &td, &ctx)?;
}
```

Rollback on failure calls `shutdown()` instead of `unregister_thing()`.

### 2.5 Update `Servient::destroy_thing`

```rust
// BEFORE:
for binding in self.server_bindings.iter() {
    binding.unregister_thing(id);
}

// AFTER:
for binding in bindings.iter() {
    binding.shutdown(id);
}
```

## Step 3 — Zenoh binding adaptation (`clinkz-wot-protocol-bindings-zenoh`)

### 3.1 Update `ZenohServerBinding::serve` (`server.rs`)

Merge the logic from `register_thing` + `configure` into `serve`:

```rust
fn serve(&self, thing_id: &ThingId, td: &Thing, ctx: &BindingContext) -> CoreResult<()> {
    // 1. Store event_broker from ctx (was configure())
    if let Ok(mut broker) = self.event_broker.lock() {
        *broker = Some(ctx.event_broker.clone());
    }
    // 2. Declare routes (was register_thing())
    // ... existing route planning + declaration logic ...
    // 3. Spawn draining task on std (closes the dead async_rx gap!)
    #[cfg(feature = "async")]
    if let Some(dispatch) = &ctx.dispatch {
        self.spawn_draining_task(Arc::clone(dispatch));
    }
    Ok(())
}
```

The draining task (the key fix for the dead `async_rx`):

```rust
#[cfg(feature = "async")]
fn spawn_draining_task(&self, dispatch: Arc<dyn Dispatch>) {
    let rx = Arc::clone(&self.async_rx);
    let server = Arc::new(self.clone());  // or a weak ref
    tokio::spawn(async move {
        loop {
            // Lock the mutex-guarded receiver, recv one, release.
            let req = match rx.lock().await.recv().await {
                Some(req) => req,
                None => break, // channel closed
            };
            let resp = dispatch.serve_request(req).await;
            server.send_response(resp);
        }
    });
}
```

### 3.2 Update `ZenohServerBinding::shutdown`

    // Was unregister_thing — undeclare routes + remove broker sinks.
    // The draining task terminates when the channel closes (routes gone
    // → no more callbacks → channel drains → recv returns None).

## Risk Analysis

| Risk | Mitigation |
|---|---|
| `Arc<dyn ClientBinding>` dispatch through `Deref` — does `binding.invoke()` work through `Arc`? | Yes; `Arc<T>: Deref<Target=T>`, and `ClientBinding` is implemented for `Box<dyn ClientBinding>` already. May need an `impl ClientBinding for Arc<dyn ClientBinding>` or remove the `Box` impl and call through `Deref`. Verify in Step 1.6. |
| Zenoh draining task lifetime — `spawn_draining_task` needs a reference to `self` (for `send_response`). | Clone the needed fields (`reply_targets`, etc.) into the task, or hold a `Weak<ZenohServerBinding>`. The binding is `Arc`-shared from the handle, so `Arc::clone` is viable. |
| `serve()` called multiple times (shared binding, multiple Things) — each call declares routes for one Thing. | Same as today's `register_thing` — the binding tracks per-Thing routes in its internal map. `serve` just replaces `configure + register_thing` in one call. |
| `shutdown()` must cancel the draining task. | The task terminates when the channel closes. After `shutdown` undeclares routes, no new callbacks fire → channel drains → `recv()` returns `None` → task exits. For immediate cancellation, a `CancellationToken` per Thing could be added (deferred optimization). |

## Sequencing Notes

This refactor is **not phase-gated** (P0–P4 are complete). It is a standalone
amendment that can land as a single PR or a short series. The recommended
sequence is Steps 1→2→3→4 in order, with each step's crate compiling in
isolation before the next begins. The workspace breaks between Step 1 and
Step 3 (core changes break servient + zenoh) — this is the accepted cost of
a trait-surface rewrite with no external consumers.
